# V2 — Harness Runtime (trabalho futuro)

Esta versão estende a V1 com validação por execução concreta de harnesses. O objetivo é cobrir padrões que o Formalizer não consegue formalizar para o ESBMC (ex: chamadas de método, slicing, padrões dinâmicos).

> **Status:** implementado no código, desabilitado por padrão. Não entra nas métricas do artigo V1.

## Diferença em relação à V1

| Aspecto | V1 | V2 |
|---|---|---|
| Verificação | Formal — BMC simbólico (ESBMC) | Concreta — execução controlada do harness |
| Prova | Formal: explora todos os inputs no bound | Evidência: um input específico reproduziu a exceção |
| Cobertura | Limitada ao que o Formalizer suporta | Maior — cobre padrões não formalizáveis |
| Confiança científica | Alta — contraexemplo formal | Média — depende da qualidade do harness gerado |

## Fluxo V2

```
arquivo Python → LLM → AST → Formalizer
                                   ↓ (não formaliza)
                            LLM gera harness
                                   ↓
                      Runtime Harness Validator
                      (execução controlada, sandbox)
                                   ↓
                            Classifica resultado
```

O harness segue o caminho de **fallback**: só é usado quando o Formalizer não gera propriedade formal. Se o Formalizer consegue, o ESBMC formal tem prioridade.

## Classificações V2

| Classificação | Descrição |
|---|---|
| `runtime_reproduced_by_harness` | Harness reproduziu a exceção esperada com as entradas fornecidas |
| `runtime_not_reproduced` | Harness executou sem levantar exceção ou levantou exceção diferente |
| `runtime_inconclusive` | Harness rejeitado por segurança, timeout ou erro de execução |

> **Importante:** `runtime_reproduced_by_harness` **não equivale** a `llm_confirmed_by_esbmc`. São formas de validação com pesos científicos diferentes. Nunca some os dois como TP na mesma métrica.

## Segurança do harness

O `runtime_harness_validator.py` executa o harness em sandbox restrito. São rejeitados:

- `import` de módulos externos
- `eval`, `exec`, `compile`
- `open`, `os.system`, `subprocess`
- loops sem bound (`while True`)

## Como habilitar (experimental)

```bash
# Habilitar harness como fallback no modo full
python src/main.py --mode full \
  --input examples/labeled/ok/bugs \
  --model gpt-4o \
  --enable-harness \
  --bound 5 --timeout 30

# Modo esbmc-harness (harnesses gerados automaticamente)
python src/main.py --mode esbmc-harness \
  --input examples/labeled/ok/bugs \
  --bound 5 --timeout 30
```
